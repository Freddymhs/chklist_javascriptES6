arrow function

  • una funcion que deleg el this a un scope superior
  • una funcion que tiene de forma implicita el return
  • mas corto que function clasica

destructuring

  • permite desempaquetar valores de tipo objecto o arreglo
  • mezclado con default params permite mucha flexibilidad

default parameter && default destructuring

  • podemos asignar parametros por defecto en props para una funcion que los requiera
  • desestructurando podemos crear un objecto en caso que no exista o ponerle un alias

rest/spread

  • spread permite dispersar propiedades ya que obj u array
  • rest es unir un grupo de propiedades en un array u objecto

consice literals

  • esto es asignar propiedades a objectos sin tener que duplicar su key y su valor

closures

  • es una funci√≥n que guarda referencias de su ambito lexico
  • podriamos asignar este closure a una varaible o muchas
  • permite no contamiar el scope con otras variables
  • permite proteger datos internos y asi no puedan ser accededidos desde fuera

promises

  • una forma de trabajar con codigo de forma Asincrono
  • existen muchas formas de tratarlos con promises all o race u otros de forma continua asegurando un orden
  • se puede usar trycatch o .catch para manejar los errores de codigo en peticiones asincronas
  • historicamente comenzo como "callbackhell" paso a -> ".then .catch" y luego a "Promises"
  • la siguiente "evolucion" es azuar sintactica

Async / await

  • esta es una evolucion para manejar promesas de forma ultra comoda
  • este necesita de try catch envolviendolo para funcionar correctamente
